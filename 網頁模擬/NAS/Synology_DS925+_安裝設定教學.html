Synology DS925+ 開箱到上線教學

新手友善・一步步引導・4bay NAS + UPS + RAID1

需求 1.開箱 2.硬碟 3.接線 4.固定IP 5.DSM 6.RAID1 7.帳號 8.Windows 9.QuickConnect 10.App 11.UPS 12.資安 13.驗收 14.QA 15.出處

📋 本次需求整理

  • 到貨設備：Synology DS925+（4-bay NAS）＋ UPS ＋ 2 顆 6TB 硬碟
  • 現有網路：中華電信 HiNet 光世代 ONT/數據機 → 自己的路由器
  • IP 規劃：未來可能對外開放服務 → 用路由器 DHCP 保留 方式固定內網 IP（不改 NAS 本身設定，較安全）
  • QuickConnect：要使用，但 ID 用隨機難猜字串加強安全性
  • 使用情境：筆電（Windows）、平板、手機都要能存取
  • 完成範圍：硬碟安裝、網路連結、DSM 初始化、RAID1、帳號密碼與資料夾、內外網資料夾分享、UPS 設定、基礎資安強化
1 開箱與內容物確認 ▾

打開 momo 箱子後，先確認以下物品是否齊全，再開始動手：

  • DS925+ 主機
  • 電源線 1 條
  • 網路線（RJ-45）通常 2 條
  • 硬碟托盤鑰匙（若為上鎖款式）
  • 快速安裝指南
  • UPS 主機、UPS 電源線、UPS 對 NAS 的 USB 訊號線
  • 2 顆 6TB 硬碟（另購，非原廠隨附）
若有缺件，先聯絡 momo 客服處理，不要急著裝機。
2 硬碟安裝 ▾
  1. NAS 完全斷電（尚未接電源線，維持這個狀態）。
  2. 按住硬碟托盤按鈕/卡榫，向外拉出空托盤（先用 Bay 1、Bay 2）。
  3. 3.5 吋硬碟：拆開托盤兩側固定面板 → 放入硬碟 → 面板裝回卡緊（免螺絲）。
  4. 2.5 吋硬碟才需要用附贈小螺絲鎖在托盤上。
  5. 裝好硬碟的托盤直線推入機身插槽，推到底聽到「喀」一聲，代表卡緊到位。
  6. Bay 3、Bay 4 先保留空著（未來擴充用）。
⚠️ 若硬碟裡本來就有資料，安裝進 NAS 建立儲存空間時會被格式化清空，請確保是全新或已清空的硬碟。
📖 出處：DS925+ 產品使用手冊－硬體元件安裝（Synology 官方）
3 接線（電源／網路／UPS） ▾
  1. UPS 先上線：電源線插到牆壁插座 → 開啟電源開關 → 確認面板顯示「有市電輸入」。
  2. NAS 電源線 → 插到 UPS 標示「電池供電＋防突波」的插座（不是每個插座都有電池支援）。
  3. 網路線：一端接 NAS 背後 LAN 孔，另一端接你自己路由器的 LAN port（不要接到 ONT）。
  4. UPS 訊號線（USB）：一端接 UPS，另一端接 NAS 背後 USB 埠。
  5. 確認接妥後，按下 NAS 電源鍵開機，等待約 1–3 分鐘至指示燈穩定。
4 路由器端設定固定 IP（DHCP 保留） ▾

目的：讓 NAS 每次開機都拿到同一個內網 IP，未來對外開放服務時不會因 IP 變動而失效，這是風險最低的做法，不需更動 NAS 本身網路設定。

  1. NAS 開機後會先跟路由器要一個「臨時」動態 IP。
  2. 瀏覽器登入路由器管理介面（通常為 192.168.1.1 或 192.168.0.1）。
  3. 找到「DHCP 用戶端清單 / Client List」，找到裝置名稱類似 DiskStation 的項目。
  4. 記下這台 NAS 的 MAC 位址。
  5. 到「DHCP 保留 / Address Reservation」頁面，新增一筆：MAC 填入剛剛記下的、指定 IP（例如 192.168.1.50）。
  6. 儲存後重新啟動 NAS 一次，確認拿到綁定的固定 IP。
💡 之後教學中的「NAS 固定 IP」都以 192.168.1.50 為範例，請換成你實際綁定的 IP。
若路由器介面找不到「DHCP 保留」選項，請先停在這一步，告知路由器品牌型號，改用 NAS 端手動設定靜態 IP 的替代方案。
5 DSM 初始化 ▾
  1. 電腦（Windows）需連上跟 NAS 同一個區網。
  2. 瀏覽器輸入 find.synology.com，或下載安裝 Synology Assistant 工具搜尋。
  3. 找到 DS925+ 後點選「設定」。
  4. 選擇「線上安裝建議版本 DSM」（過程中自動重開機 1–2 次，中途絕對不要拔電源）。
  5. 建立管理員帳號：帳號名稱自訂（不要用預設 admin），密碼 12 碼以上並混合大小寫、數字、符號。
  6. 幫 NAS 命名，例如 HomeNAS。
  7. QuickConnect 設定畫面先「略過」，第 9 節再回來做強化設定。
📖 出處：Synology 產品下載中心－DS925+ DSM 韌體與文件
6 建立 Storage Pool + RAID1 ▾
  1. 開啟「Storage Manager（儲存空間管理員）」。
  2. 左側選「儲存空間集區」→「新增」。
  3. 選擇「自訂」→ RAID 類型選 RAID1。
  4. 勾選 2 顆 6TB 硬碟 → 下一步 → 確認 → 套用。
  5. 系統跑「同位檢查（Parity Check）」，可能需數小時，背景執行，NAS 仍可正常使用。
  6. 建立「磁碟區（Volume）」，檔案系統選 Btrfs（支援快照、自我修復檢查）。
  7. 完成後應顯示「儲存空間集區：正常」、「磁碟區：正常」。
📖 出處： 建立儲存集區 ／ 建立並管理 RAID 儲存空間
7 使用者帳號與共用資料夾 ▾
  1. 「控制台 → 使用者與群組」→ 新增使用者：建立日常使用帳號（管理員帳號只在做系統設定時使用）。
  2. 「控制台 → 共用資料夾」→ 新增資料夾，例如 Public（可共用）、Private（個人資料）。
  3. 每個資料夾點選「編輯 → 權限」，逐一設定哪些帳號可以「讀取」或「讀寫」。
8 Windows 筆電內網存取 ▾
  1. 開啟檔案總管，網址列輸入 \\192.168.1.50（換成實際固定 IP）。
  2. 輸入 NAS 使用者帳號密碼。
  3. 右鍵點選資料夾 →「對應網路磁碟機」→ 選磁碟代號（如 Z:）→ 勾選「登入時重新連線」。
9 QuickConnect 外網存取（強化別名） ▾

QuickConnect 讓你在外面也能連回家裡的 NAS，不需要在路由器開 Port Forward，相對安全。

  1. 「控制台 → QuickConnect」→ 勾選「啟用 QuickConnect」。
  2. 使用現有 Synology 帳戶登入，或註冊新帳戶。
  3. 自訂 QuickConnect ID：
    • ❌ 不要用姓名、公司名稱、NAS 用途（例如 chou-home-nas）
    • ✅ 改用無意義隨機英數組合，例如 x7k2mq9n4（10 碼以上，混大小寫＋數字）
    • 規則：只能用英文字母、數字、連字號「-」，須以字母開頭
  4. 套用後，從外部瀏覽器輸入 https://你的ID.quickconnect.to 即可連線登入。
  5. 進階選項（「控制台 → QuickConnect → 進階」）：不要勾選「自動建立連接埠轉送規則」。
未來若要正式對外開放服務，建議屆時另外規劃固定 DDNS＋反向代理＋SSL 憑證，或改用 VPN 方式進入內網，這牽涉防火牆規則變更，屬於進階項目，不建議現在貿然開 Port。
📖 出處：QuickConnect（Synology 知識中心）
10 手機／平板 App 設定 ▾

iOS（iPhone / iPad）

  1. App Store 搜尋安裝：DS file
  2. 開啟 App →「新增伺服器」
  3. 家中 WiFi（內網）時：輸入固定 IP，例如 192.168.1.50
  4. 外面（外網）時：改選「QuickConnect」，輸入你的 QuickConnect ID
  5. 輸入 NAS 帳號密碼登入即可瀏覽／上傳／下載

Android（手機/平板）

  1. Google Play 搜尋安裝：DS file
  2. 步驟同 iOS：新增伺服器 → 內網固定 IP 或 QuickConnect ID → 帳密登入

筆電自動同步（選用）

  1. Synology 官網下載「Synology Drive Client」安裝在筆電
  2. 輸入固定 IP 或 QuickConnect ID → 登入 → 選擇要自動同步的資料夾
11 UPS 設定 ▾
  1. 「控制台 → 硬體與電源 → 不斷電系統」。
  2. 接線正確時會顯示紅字提醒「不斷電系統已連接」，勾選「啟動不斷電系統支援」。
  3. UPS 類型選擇「USB 不斷電系統」。
  4. 設定「進入安全模式前的等待時間」：家用建議 3–5 分鐘。
  5. 若要讓其他電腦查詢 UPS 狀態，可勾選「啟用網路不斷電系統伺服器」（進階選用，非必要）。
  6. 建議做一次斷電模擬測試：手動拔掉市電插頭，確認 NAS 依設定時間安全關機；插回市電後確認能自動開機。
📖 出處：不斷電系統（Synology 知識中心）
12 基礎資安強化 ▾
  1. 「控制台 → 更新與還原」→ 確認 DSM 為最新版本。
  2. 管理員帳號啟用「雙步驟驗證（2FA）」。
  3. 「控制台 → 安全性 → 帳號」→ 啟用「自動封鎖」。
  4. 「控制台 → 安全性 → 防火牆」→ 啟用防火牆，只開放必要服務連接埠。
  5. 執行一次「Security Advisor」健檢，依建議修正紅色/黃色警示項目。
  6. 停用不需要的服務（FTP、Telnet、SNMP 等保持關閉）。
📖 出處：Security Advisor（Synology 知識中心）
13 完工驗收清單 ▾
  • 路由器 DHCP 清單顯示 NAS IP 固定，重開機後 IP 不變
  • Storage Manager 顯示「正常」，RAID1 容量約 6TB 可用
  • Windows 筆電可透過固定 IP 正常存取共用資料夾
  • 手機（切到行動網路）用 QuickConnect 隨機別名登入成功
  • 平板同樣測試成功
  • 「不斷電系統」頁面顯示 UPS 已連接、狀態正常
  • 斷電模擬測試：安全關機＋市電恢復後自動開機
  • Security Advisor 檢查無高風險項目
  • 管理員帳號已開啟雙步驟驗證
14 延伸 QA ▾
Q1：只裝 2 顆硬碟做 RAID1，之後想擴充怎麼辦？
之後可直接在 Bay 3、Bay 4 插入新硬碟，於 Storage Manager 建立新的儲存空間集區，與現有 RAID1 分開管理、互不影響。
Q2：RAID1 是不是就不用備份了？
不是。RAID1 只防「單顆硬碟故障」，無法防範誤刪、勒索病毒、天災。建議日後另外規劃 Hyper Backup 備份到外接硬碟或雲端。
Q3：QuickConnect 會不會比較慢？
走中繼服務（Relay）確實會較慢，換取的是不用開 Port、曝露面較小。
Q4：忘記固定 IP 或 QuickConnect ID 怎麼辦？
固定 IP 查路由器 DHCP 保留清單；QuickConnect ID 登入 account.synology.com 查看已註冊裝置。
Q5：DSM 更新會不會把資料洗掉？
一般不會清空 Volume 內容，但建議更新前先確認沒有重要作業進行中，並定期備份。
Q6：手機 App 一定要用 QuickConnect 嗎？
不一定。家中 WiFi 可直接用固定 IP，速度較快，只有外出才需要 QuickConnect。
Q7：以後真的要對外開放給客戶用，現在設定夠嗎？
目前屬家用等級安全做法。正式對外建議屆時規劃固定 DDNS、反向代理、SSL 憑證或 VPN，並重新檢視防火牆規則。
